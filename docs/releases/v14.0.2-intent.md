# Release intent — v14.0.2

## What this release is about

This patch release fixes Webpack 4 compatibility with the package.json exports map. The wildcard pattern (`./types/*`) in exports is not fully supported by Webpack 4, causing module resolution failures for Cedar v17 consumers. This release adds explicit exports for the type paths used by Cedar, ensuring compatibility with both Webpack 4 and modern bundlers.

## What consumers will care about most

- Webpack 4 compatibility: Cedar v17 consumers using Webpack 4 can now upgrade without module resolution errors
- No API changes: This is a purely internal package.json exports fix
- Backward compatible: Modern bundlers continue to work with both explicit exports and the wildcard fallback

## What changed internally that should NOT appear in release notes

- Added explicit exports for type paths: breakpoint, color-background, color-border, prominence, radius, space, space-scale, and their respective .keys variants
- Kept wildcard exports as fallback for modern bundlers
- No changes to token values, types, or build outputs

## Known breaking changes

None. This is a backward-compatible patch.

## Cross-repo notes

This release is required for Cedar v17.0.1 consumers using Webpack 4. Cedar v17.0.1 will recommend @rei/cdr-tokens v14.0.2 for Webpack 4 compatibility.

## BOLO — things consumers should watch for

None.

## Success criteria

- Webpack 4 consumers can successfully import from `@rei/cdr-tokens/types/*` paths
- No regression for modern bundlers (Vite, Webpack 5, Rollup)
- Cedar v17.0.1 builds successfully with Webpack 4
